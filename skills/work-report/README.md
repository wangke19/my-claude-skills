# Work Report Generator Skill

Automatically generate comprehensive work reports by orchestrating Jira and GitHub MCP servers.

## Quick Start

```
User: "generate daily report"
User: "generate weekly report"
User: "generate quarterly report"
User: "generate yearly report"
```

## What It Does

This skill:
1. Fetches your Jira issues from the specified time period
2. Fetches your GitHub pull requests from the same period
3. Combines the data and generates three report formats:
   - **Full Report** (Markdown) - Professional format with all details
   - **Brief Report** (Text) - Quick standup format for Slack/email
   - **CherryTree** (XML Database) - Historical tracking with rich text

## Report Types

| Type | Time Period | Use Case |
|------|-------------|----------|
| **Daily** | Today only | Daily standups, end-of-day summaries |
| **Weekly** | Last 7 days | Weekly status updates, sprint reviews |
| **Quarterly** | Last 90 days | Quarterly reviews, performance tracking |
| **Yearly** | Last 365 days | Annual retrospectives, year-in-review |

## Output Location

```
~/work/work-reports/
├── 2026/
│   └── 02/
│       └── Week-06/
│           ├── daily_report_2026-02-06.md         ← Full report
│           └── daily_report_2026-02-06_brief.txt  ← Brief report
└── daily_report.ctd                                ← CherryTree database
```

## Prerequisites

**Required MCP Servers:**
1. **Jira MCP Server** - For fetching Jira issues
2. **GitHub MCP Server** - For fetching GitHub PRs
3. **Work Report Generator MCP Server** - For formatting reports

All three must be running and authenticated.

## Usage Tips

### For Daily Standups

1. Run `generate daily report` at end of workday
2. Open the brief report: `~/work/work-reports/2026/02/Week-06/daily_report_2026-02-06_brief.txt`
3. Copy content to Slack/email
4. Done in 2 minutes!

### For Weekly Updates

1. Run `generate weekly report` every Friday
2. Review the full report for complete details
3. Use for status emails or sprint reviews

### For Performance Reviews

1. Run `generate quarterly report` before review meetings
2. Open CherryTree database to review long-term trends
3. Use statistics for self-evaluation

## Configuration

### Optional: Enable Encrypted CherryTree

```bash
export CHERRYTREE_ENCRYPTED="true"
export CHERRYTREE_PASSWORD="your-secure-password"
```

This encrypts the CherryTree database file using 7-zip AES-256 encryption.

## Troubleshooting

### No Jira data?

- Check Jira MCP server is running
- Verify `JIRA_USERNAME` and `JIRA_API_TOKEN` environment variables
- Test: Try calling `mcp__jira__jira_search` directly

### No GitHub data?

- Check GitHub MCP server is running
- Verify GitHub authentication (usually via `gh` CLI)
- Test: Try calling `mcp__github__search_pull_requests` directly

### Report generator fails?

- Check work-report-generator MCP server is running
- Start it: `cd ~/work/gitlab/work-report-generator && python src/server.py`

### Empty report?

- Normal if you had no activities in the specified time period
- Verify by checking Jira and GitHub directly
