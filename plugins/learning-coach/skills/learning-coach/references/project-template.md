# End-to-End Project Template

Use this to cap a learning path with a real, deliverable project.

## Goal

Define what the learner will ship:
- Concrete artifact (CLI tool, API, dashboard, etc.)
- Who would use it and why
- What "done" looks like

## Steps

### 1. Setup
- Environment, dependencies, project scaffold
- Verify everything works before writing logic

### 2. Implement
- Build the core feature incrementally
- Each increment is a working, testable state

### 3. Test
- Write tests for the core behavior
- Cover at least one happy path and one failure case

### 4. Debug
- Introduce a realistic bug or edge case
- Apply the debug lab workflow to fix it

### 5. Optimize
- Identify one performance or reliability improvement
- Measure before and after

### 6. Production Checklist
- [ ] Error handling for external inputs
- [ ] Logging/observability in place
- [ ] Configuration externalized (no hardcoded values)
- [ ] README with setup and usage instructions
- [ ] At least basic test coverage
