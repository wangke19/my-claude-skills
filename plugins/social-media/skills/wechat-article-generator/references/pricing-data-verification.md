# 大模型 API 价格数据核实规范

## 核心原则

**大模型 API 价格和运营商套餐数据不能直接信任，必须主动核实。**
- API 价格由各厂商随时调整，文章中的历史数据极易过时
- 运营商套餐的"词元量"标注混乱（"万词元"、"1000万"混用），单价必须用公式换算

## 运营商词元套餐：单价换算公式

```
月费（元）÷ 词元量 × 100 = 元/百万token
```

注意：运营商词元量单位不统一，需统一转换为"百万token"再计算：

| 运营商 | 套餐 | 月费 | 词元量（原始） | 词元量（百万） | 单价（元/百万token） |
|--------|------|------|---------------|--------------|-------------------|
| 中国电信 | 轻享版 | 9.9元 | 1000万 | 10 | 9.9÷10=**0.99** |
| 中国电信 | 尊享版 | 49.9元 | 8000万 | 80 | 49.9÷80=**0.62** |
| 中国移动 | 算力月包 | 24.9元 | 1000万 | 10 | 24.9÷10=**2.49** |
| 中国联通 | 个人版 | 15元 | 600万 | 6 | 15÷6=**2.50** |
| 上海电信 | 按量 | 1元 | 25万 | 0.25 | 1÷0.25=**4.00** |

⚠️ **易错点**：月费数字往往看起来很便宜（如"9.9元"），但词元量单位可能是"万"。直接用月费数字当单价会严重低估价格（如把0.99写成9.9，误差10倍）。

## 图表单位规范

柱状图X轴标签：**"元/百万token（越短越划算）"**

- 柱形越短 = 价格越低 = 越划算
- 错误写法：标"百万/元"（含义相反，"越高越划算"）
- 柱形排序：价格从低到高（最短 → 最长）

正确排序示例（从短到长）：
1. 通义千问Flash 0.2 → 最短
2. 电信轻享版 0.99
3. DeepSeek V4-Flash 1.0
4. 移动算力月包 2.49
5. 联通个人版 2.50
6. GLM-5 / 上海电信 / Kimi K2.5 → 最长（4.0）

## 核实流程（发布前必做）

1. **搜索核实**：对每个模型用 `site:官网 domain 价格` 搜索
2. **提取验证**：用 `web_extract` 访问定价页面，确认数字
3. **换算运营商**：用公式逐一计算运营商套餐单价
4. **图表标注**：数据截止日期 + "价格变动风险请以实际为准"

## 常见错误记录

| 错误 | 后果 | 教训 |
|------|------|------|
| 直接用月费数字当单价 | 运营商套餐价格被低估10倍 | 必须÷词元量÷100 |
| 引用非官网价格 | 与官网不符，失信读者 | 用 web_extract 查官网 |
| 图表标"越高越划算" | 单位与文字含义矛盾 | 用"元/百万token，越短越划算" |
| 通义千问写区间0.05~0.25 | 已改为固定0.2元 | 查官网最新定价 |
