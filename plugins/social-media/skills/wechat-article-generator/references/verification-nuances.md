# AI 产品声量验证：多平台交叉核对要点

> 来源：2026年5月 session — 验证"OpenClaw 小龙虾声量降到冰点"claim

## 核心教训

**错误做法**：只查 GitHub commit 活动和 Stars 绝对值 → 得出"声量未降"结论
**正确做法**：区分评估维度，不同平台的数据趋势可能完全相反

## OpenClaw 声量验证结果（2026年5月）

| 维度 | 趋势 | 核实状态 |
|---|---|---|
| GitHub Stars | 36万 → 37.3万（仍在涨，速度放缓） | ✅ 属实，但"Star 停滞"表述不准确 |
| GitHub commit/Release | 5月17日仍有 v2026.5.17 第162个版本 | ✅ 活跃 |
| 微信指数 | 从峰值下降 90%+ | ✅ 新浪财经3月22日报道可查 |
| 中文社媒讨论 | 3月爆火 → 4月断崖 → 5月冰点 | ✅ 每日经济新闻、东方财富等可查 |
| 媒体报道调性 | 从"爆火封神"转向"复盘/劝退" | ✅ 趋势明确 |
| 30天卸载率/用户留存 | 三分之一用户最终没留下 | ⚠️ 来自调研报告，非官方披露 |
| 微信官方插件 | 2026年3月22日腾讯上线 ClawBot 插件 | ✅ 知乎文章可查 |

## 验证 AI 产品声量 claim 的标准清单

1. **GitHub 绝对值** ≠ 社媒声量（两个指标解耦）
2. **中文平台**：微信指数、知乎文章标题/日期、公众号发文频率
3. **媒体叙事转变**：从"爆火"到"泡沫/劝退"的时间节点
4. **用户留存 vs 初期热度**：初期爆发≠长期留存
5. **官方动作 vs 舆论热度**：腾讯发插件说明商业价值存在，即使大众讨论已退潮

## 关键陷阱

- 用"GitHub 还在更新"否定"声量降到冰点"——这是两类指标
- 依赖单一搜索结果（第一次我搜到"25万 stars"就下结论）
- 未区分"技术圈持续活跃"和"大众讨论退潮"的不同现象
